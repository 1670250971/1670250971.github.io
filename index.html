ubuntu终端命令

1、文件与目录

分区：Linux没有分区概念

Linux以目录的组织，只有根目录/

/bin ：可执行文件

/sbin: 供超级用户使用的可执行文件

s=super

/etc: 配置文件

eg: 源地址： /etc/apt/sources.list

/home: 家目录，给用户存储文件用

-每一个用户都有一个用户名命名的文件夹

-收藏夹中的主目录=用户自己的家目录

-同时可以看收藏夹中也有家目录各子目录的快捷方式

/mnt: 挂载设备用

/boot: 引导程序

操作系统：内核->系统调用->内置的命令行或基本的图形程序->第三方应用->用户

二、Linux终端与命令

1、ls

-l :详细输出文件或目录信息

-h :人性化方式显示文件大小

Bytes<KB< MB <GB <TB <PB<EB <ZB<YB

-a :显示隐藏文件（以.开始）

.： 当前目录

.. : 上一级目录

~： 表示用户自己的家目录

= /home/username

ls -a -h = ls -al

2、 pwd

print working directory

pwd : 显示当前的工作目录

3、cd 切换文件夹

change directory

cd /

cd ~

cd ..

Linux中的命令区分大小写

4、touch

格式：touch 文件名

如果文件不存在则新建一个

如果文件存在则修改最近修改时间

5、mkdir

创建目录make directory

支持同时创建多个文件夹：

-同级目录，用空格间隔 mkdir a b

-层级目录，用-p参数 mkdir -p a/b/

同级目录下，文件与文件夹不能重名

6、rmdir

删除空目录remove directory

终端中删除的内容不进回收站

7、rm

删除文件remove

可以使用通配符：

–*:任意多字符

–?:一个字符

rm -r 非空目录名： 删除非空目录

终端中删除的内容不进回收站！！

8、clear

清屏

9、技巧：

（1)利用上下键

(2)使用Tab: 命令与路径自动补全

(3)区分相对路径与绝对路径：

-绝对路径：从/开始的路径

因为=/home/username，所以开头的也称为绝对路径

-相对路径：从当前目录开始的路径

一、SSH配置免密码登录

1、原理：

a.生成一个密钥对，在客户机上存放私钥，在服务器上存放公钥

b. 每次登录时，取私钥并与服务器上的公钥配对，配对成功表示认证通过

2、步骤

Q1: 如何生成？

ssh-keygen(key generator): 根据机器的特征值生成id_rsa，id_rsa.pub

Q2：生成后的密钥存放在何处？

存放在用户目录下的.ssh

Q3：服务器如何配置

任务：id_rsa.pub文件的内容添加到远程主机.ssh目录下的authorized_keys文件

01、Linux系统中

ssh-copy-id [-p port] username@host

02、Windows

a. 可视化方式复制

b. scp: secure copy，用于远程的安全拷贝

scp [-P port] 本地文件 username@host：Desktop/01.py

cat id_rsa.pub >~/.ssh/authorized_keys

SCP扩展

1、本地文件上传至服务器

2、远程文件下载到本地

任务：将远程主机Desktop下的hello.py下载

scp -P 22 lester@192.168.56.1:Desktop/hello.py hello_world.py

3、将本地文件夹上传至服务器

scp -r zjnu lester@192.168.56.1:Desktop

4、将远程文件夹下载至本地

scp -r lester@192.168.56.1:Desktop/01 01_copy

1、Git

版本控制系统（Version Control System, VCS）

其它VCS: CVS, Subersion, Visual source safe

开源共享，可以自架服务器，同时使用客户端访问与应用

专用的第三方的Git服务，相当于租用了别人的Git服务器，在上面创建软件仓库并进行版本管理。

典型的Git服务：Github，Gitlab

2、Git中的基本概念

-仓库：相当于一个托管的单位，一般对应一个项目

-版本： 仓库中的文件的某一个状态

3、Git安装

Ubuntu或者Raspbian OS：

apt-get/apt install git

4、Git的身份认证

一般情况下采用SSH或PGP密钥认证的方式。

参考：SSH的免密登录

5、创建Git仓库

https://github.com/ 注册一个账号

Repositories（仓库）中选择New

仓库地址：

https://github.com/scratchzjnu/zjnumaker.git

git@github.com:scratchzjnu/zjnumaker.git

6、Git初始化配置

提前准备用户名、邮箱：scratchzjnu, service@ilester.net

git config –global user.name “用户名”

git config –global user.email “邮箱”

ssh-keygen -t rsa -C “邮箱”

cat ~/.ssh/id_rsa.pub并将输出的内容添加到Github账户/setttings/ssh and gpg keys

7、初始化仓库

创建一个用作本地存储的仓库目录

git init（会初始化仓库，相关配置在.git下）

注意：.git文件尽量不要修改

8、将文件添加到仓库并提交

前置：创建一个文件，例如：readme.md

git add readme.md

git commit -am “开发日志”

git status

9、查看文件的版本记录

git log 文件名查看

commit 后面的标识即为版本号

10、回退版本

git checkout 版本号 文件名

git

11、将文件上传到Git服务器

前置：需要找到Git仓库地址，配置好SSH或GPG密钥

git remote add origin github上的地址

12、通常操作

单用户：

本地修改文件

git add

git push

要求：项目名称均为 用户名.github.io

权限更改，用户名组名添加

一、Recap: 权限

r:读

w:写

x：执行

1:类型

后9位每3个一组

第1组：所有者，RWX

第2组：所在组,RWX

第3组：其它用户,RWX

rw-r–r–: 110 100 100

rwxr-xr-x: 111 101 101

最大的是：7

最小的是：0

rwxrwxrwx: 777 （Web程序的上传目录）

———–：000

只读：R–,100，4

读写：RW-,110，6

读取与执行：R-X，101，5

所有权限：7

a, chmod的两种方法

chmod +/- rwx filename/directory

chmod mask filename/directory

b, 权限掩码以及原理

groupdel:

chgrp -R 组名 目录名

4、添加用户

别名登录，用户权限命令

1、别名登录

给我们访问的远程主机设置一个容易记性的别名

类似于：域名VS IP地址

不同点：域名需要在DNS注册，别名在本机注册

Windows方法：

（1).ssh下新建文件config

（2）config中配置别名

Host ubuntu

HostName 192.168.56.1

User lester

Port 22

（3）别名登录：ssh user@别名

2、用户权限

ls-l为例

lester@ubuntu:~/Desktop$ ls -lh

总用量 28K

drwxr-xr-x 4 lester lester 4.0K 3月 11 09:51 01

-rw-rw-r– 1 lester lester 62 4月 1 09:45 config.txt

-rw-r–r– 1 lester lester 599 3月 11 10:17 def.txt

-rw-r–r– 1 lester lester 779 3月 11 10:10 hello.py

-rw-rw-r– 1 lester lester 405 3月 25 09:36 id_rsa.pub

-rw-r–r– 1 lester lester 599 3月 11 10:19 world.py

drwxrwxr-x 3 lester lester 4.0K 3月 25 09:49 zjnu

依次：权限标识符、硬链接数、用户与组、文件容量、最后修改日期、名称

d rwx rwx r-x

（1)权限标识符：10位

第一位：d/- （类别，d表示目录-表示文件）

后九位：

-第一组RWX：拥有者的权限

-第二组RWX：所在组的权限

-第三组RWX：其它人的权限

R: 可读

W：可写

X:可执行

（2)chmod

格式： chmod +/- rwx 文件名

示例：文件夹的可执行权限

示例：文件加可执行权限（Python脚本、shell脚本）

3、超级用户

Linux用户类型：

-root用户：用于系统维护，具有所有权限

-绝大多数的Linux发行版不推荐使用root

-安装时创建的为标准用户

sudo

-su: substitude user替换用户

-预设的身份是root

-需要输入密码，有5分钟有效期

-如果未经授权尝试使用root,会发送警告
